# 悟道 (Wudao) — UI 设计指南

> 本文档为专业 UI 设计师准备的完整设计指南。
> 目标：为一个**个人交易执行工作台**设计一套专业、克制、信息密度高的界面系统。

---

## 1. 产品定位与用户画像

### 产品是什么
悟道是一个**面向单人交易者（Jason）的本地优先交易执行工作台**。它不是看盘软件，不是券商 App，也不是量化研究平台。

它的核心价值是：
- 统一手动/半手动交易的执行、风控、仓位管理、记录、复盘
- 强制纪律检查（没止损不能下单、超风控限制不能下单）
- 把交易行为放进可审计、可追溯的系统中

### 用户画像
- **单人使用**：不需要多用户、权限系统、协作功能
- **技术背景**：用户是程序员 + 交易者，偏好信息密集的专业界面，厌恶花哨动画和过度留白
- **使用场景**：每天在 macOS 浏览器中打开，同时可能开着 TradingView、交易所页面、终端
- **核心诉求**：打开后 10 秒内知道当前系统状态（账户、持仓、风险、待处理事项）

### 设计气质
- **Bloomberg Terminal meets 现代 SaaS Dashboard**
- 专业、克制、高信息密度
- 深色主题为主（交易软件惯例，减少视觉疲劳）
- 关键数据一眼扫到，不需要点开层层折叠面板
- 数字排列整齐，金融数据使用等宽/半等宽字体

## 2. 设计系统基础

### 2.1 色彩体系

**主题**：深色模式为默认，可切换浅色模式

| Token | 用途 | 深色值参考 |
|-------|------|-----------|
| `background` | 全局底色 | `#0a0a0f` ~ `#111118` |
| `card` | 面板/卡片底色 | `#16161d` ~ `#1a1a24` |
| `card-elevated` | 浮层/弹窗 | `#1e1e28` |
| `border` | 分割线/边框 | `#2a2a36` |
| `text-primary` | 主要文字 | `#e8e8ed` |
| `text-secondary` | 次要文字/标签 | `#8b8b9a` |
| `text-muted` | 提示/占位文字 | `#5a5a6e` |

**语义色**：

| Token | 用途 | 值参考 |
|-------|------|--------|
| `profit` / `long` | 盈利/多头 | `#22c55e` 绿色 |
| `loss` / `short` | 亏损/空头 | `#ef4444` 红色 |
| `warning` | 警告/注意 | `#f59e0b` 琥珀色 |
| `info` | 信息/中性 | `#3b82f6` 蓝色 |
| `accent` | 品牌强调色 | `#8b5cf6` 紫色（或 `#6366f1` 靛蓝） |
| `sandbox-badge` | Sandbox 模式标识 | `#6366f1` 靛蓝 |
| `live-badge` | Live 模式标识 | `#ef4444` 红色（危险感知） |

### 2.2 字体

- **界面字体**：Inter / SF Pro（系统 UI）
- **数字/金融数据**：JetBrains Mono 或 IBM Plex Mono（等宽，保证数字列对齐）
- **中文辅助**：Noto Sans SC 或 PingFang SC

### 2.3 圆角与间距

- 卡片圆角：`8px`
- 按钮圆角：`6px`
- 小组件（Badge）：`4px`
- 间距基数：`4px` 倍数（4, 8, 12, 16, 24, 32）

### 2.4 信息密度原则

- 这不是消费级 App，**不需要大面积留白**
- 对标 Bloomberg Terminal / TradingView 的信息密度
- 表格行高紧凑（32-36px），卡片间距适度（8-12px gap）
- 数字右对齐，带颜色标识正负
- 百分比变化用 `+2.34%` / `-1.56%` 格式，配红绿色

### 3.2 TopBar

**高度**：56px，固定在顶部

**左侧**：
- 当前页面标题（如 "Dashboard"、"Positions"）

**右侧**（从右到左）：
- **模式徽章**：`SANDBOX` (靛蓝底，白字) 或 `LIVE` (红色底，白字，可加微闪动效果)
  - 这是安全关键信息，必须始终可见，字号不能太小
  - LIVE 模式建议加一个微妙的脉动光晕效果，强化"这是真钱"的感知
- **通知铃铛图标**：带未读数量角标
- **系统状态指示器**：小圆点表示 API/Exchange 连接状态（绿=正常，黄=延迟，红=断开）
- **当日盈亏摘要**：`Today P&L: +$234.50` 或 `-$87.20`（带红绿色）

### 3.3 Sidebar

**宽度**：224px，固定不可折叠（MVP 简化）

**顶部**：品牌标识
- 中文 "悟道" + 英文 "Wudao"
- 可考虑一个简洁的太极/道家风格小图标

**导航项**（图标 + 文字）：
1. **Dashboard** — `LayoutDashboard` 图标
2. **Positions** — `TrendingUp` 图标
3. **Orders** — `ShoppingCart` 图标
4. **Pending Intents** — `ClipboardList` 图标 + 未处理数量角标
5. **Journal** — `BookOpen` 图标
6. **Alerts** — `Bell` 图标 + 未读告警数量角标
7. **Settings** — `Settings` 图标

**当前选中状态**：左侧 3px 竖条强调色 + 背景色变化

**底部**（sidebar 底部固定）：
- 当前账户名称缩写
- 连接状态小圆点

## 4. 页面设计详细

### 4.1 Dashboard（首页/总览）

**核心目标**：打开后 10 秒内知道所有关键信息

**布局建议**：

```
┌────────────────────────────────────────────┐
│  Account Summary Bar（横向统计卡片带）         │
│  [净值] [余额] [未实现P&L] [已实现P&L] [风险]  │
├────────────────────┬───────────────────────┤
│                    │                       │
│  持仓概览            │  待处理事项             │
│  (Position Cards)  │  (Pending Actions)    │
│                    │                       │
├────────────────────┼───────────────────────┤
│                    │                       │
│  未成交订单          │  最近信号              │
│  (Open Orders)     │  (Recent Signals)     │
│                    │                       │
├────────────────────┴───────────────────────┤
│  风控告警带（条件性显示）                       │
│  [Risk Alerts - 红/黄色横幅]                  │
└────────────────────────────────────────────┘
```

**Account Summary Bar**：
- 5 个统计卡片横排
- 每个卡片包含：指标名称（小字灰色）、数值（大字白色/红绿色）、变化百分比（小字）
- 卡片内容：
  - **账户净值** (Account Equity): `$12,345.67`
  - **可用余额** (Available Balance): `$8,901.23`
  - **未实现盈亏** (Unrealized P&L): `+$234.50` (绿) 或 `-$87.20` (红)
  - **已实现盈亏** (Realized P&L Today): `+$156.80`
  - **总风险暴露** (Risk Exposure): `23.4%` — 如果超过阈值显示黄/红色

**持仓概览区域**：
- 最多显示 5 个当前持仓的摘要卡片
- 每个卡片：币种对、方向（LONG/SHORT badge）、数量、当前盈亏（%和$）、入场均价、当前价
- 点击跳转到 Positions 页面详情

**待处理事项区域**：
- 列出所有 `pending_review` 状态的 OrderIntent
- 每条显示：来源图标（TradingView / 手动）、币种对、方向、数量、风控状态 badge
- 快速操作：[审批] [拒绝] 按钮

**未成交订单区域**：
- 简化的表格视图：币种对、方向、价格、数量、类型、时间
- 快速撤单按钮

**最近信号区域**：
- 最近 5 条 TradingView 信号
- 每条显示：时间、来源、币种对、方向、策略标签、状态 badge

**风控告警带**：
- 仅在有告警时显示
- 黄色背景 = 警告（接近限额）
- 红色背景 = 严重（已触发限制，如当日最大亏损已达到）
- 内容示例："Daily loss limit reached (-$500.00). New positions blocked."

### 4.2 Positions（持仓管理）

**核心目标**：看清每一个持仓的状态、风险、止盈止损结构

**布局**：

**上方**：持仓汇总条
- 总持仓数 | 多头仓位数 | 空头仓位数 | 总未实现盈亏 | 总风险暴露%

**主区域**：持仓卡片列表（每个持仓一张卡片）

**单个持仓卡片设计**：

```
┌───────────────────────────────────────────┐
│ BTC/USDT          LONG          +12.34%   │
│ ─────────────────────────────────────── │
│ Quantity    0.5 BTC                        │
│ Avg Entry   $64,200.00                     │
│ Mark Price  $72,120.00                     │
│ Unrealized  +$3,960.00                     │
│ Risk Exp.   18.2%                          │
│ ─────────────────────────────────────── │
│ SL: $62,000  │  TP: $78,000  │ Trailing: ON│
│ ─────────────────────────────────────── │
│ [Reduce 25%] [Reduce 50%] [Close All]      │
└───────────────────────────────────────────┘
```

卡片内的关键元素：
- **头部**：币种对 + 方向 Badge (LONG=绿色, SHORT=红色) + 总收益率%
- **数据区**：等宽字体数字列，标签左对齐，数字右对齐
- **止损止盈区**：显示当前 SL/TP 设置，Trailing Stop 开关状态
- **操作区**：减仓/平仓按钮。平仓按钮用红色/橙色强调

**空状态**：
- "No open positions" + 插画或图标
- 可选引导文字："Create an order intent or wait for TradingView signals"

### 4.3 Orders（订单管理）

**布局**：Tab 切换

**Tab 1 — Open Orders（未成交订单）**
- 表格视图
- 列：时间 | 币种对 | 方向 | 类型(Market/Limit) | 价格 | 数量 | 状态 | 操作(撤单)
- 状态 Badge：`open` (蓝), `partially_filled` (琥珀), `pending` (灰)

**Tab 2 — Order History（历史订单）**
- 同样的表格结构
- 增加：成交均价 | 手续费
- 状态 Badge：`filled` (绿), `cancelled` (灰), `rejected` (红)
- 支持时间范围过滤 + 币种对搜索

**Tab 3 — Fills（成交明细）**
- 表格：时间 | 订单ID | 币种对 | 方向 | 成交价 | 数量 | 手续费

**表格交互**：
- 行 hover 高亮
- 点击行展开详情面板（显示完整订单链路：Signal → Intent → Order → Fill）
- 列头可排序
- 每页 25/50/100 条分页

### 4.4 Pending Intents（待确认意图）

**核心目标**：这是风控与纪律检查的核心界面。交易者在这里审批或拒绝交易意图。

**布局**：卡片列表，每个 OrderIntent 一张卡片

**单个 Intent 卡片设计**：

```
┌──────────────────────────────────────────────────────┐
│ ⚡ TradingView Signal          2024-01-15 14:23:05   │
│ ──────────────────────────────────────────────────── │
│ BTC/USDT  ▲ BUY   Limit @ $64,200.00                │
│                                                       │
│ Quantity:     0.5 BTC                                 │
│ Stop Loss:   $62,000.00   (-3.4%)                     │
│ Take Profit: $72,000.00   (+12.1%)                    │
│ Risk Amount: $1,100.00    (1.0% of equity)            │
│ Strategy:    breakout-v1                              │
│ ──────────────────────────────────────────────────── │
│ 🟢 Risk Check                                         │
│   ✅ Stop loss present                                │
│   ✅ Risk per trade ≤ 2% ($1,100 < $2,400)           │
│   ✅ Position size within limit                       │
│   ✅ Daily loss limit not reached                     │
│   ⚠️ No trade plan attached (warning only)            │
│ ──────────────────────────────────────────────────── │
│ Review Note: [                                    ]   │
│                                                       │
│        [Reject]                    [Approve & Submit]  │
└──────────────────────────────────────────────────────┘
```

**关键设计要素**：
- **来源标识**：图标区分来源（TradingView 闪电图标 / 手动铅笔图标）
- **风控检查结果**：每条规则单独显示通过(✅)/失败(❌)/警告(⚠️)
  - 如果任何一条为 ❌（CRITICAL），Approve 按钮禁用，显示为灰色
  - 如果有 ⚠️ 但无 ❌，Approve 按钮可用但显示警告提示
- **Review Note**：文本输入框，供交易者写理由
- **按钮设计**：
  - Reject = 灰色/中性按钮
  - Approve & Submit = 绿色/强调色按钮
  - 如果风控不通过，按钮变为 `Force Submit (Override)` + 红色按钮 + 需要额外确认弹窗

**空状态**：
- "No pending intents. All caught up."
- 可选：显示上一个已处理 intent 的摘要

### 4.5 Journal（交易日志 & 复盘）

**核心目标**：记录交易、补充复盘备注、查看交易链路

**布局**：

**左栏（列表）**：交易记录列表
- 每条记录显示：日期 | 币种对 | 方向 | 盈亏$ | 盈亏% | 策略标签 | 状态Badge
- 过滤器：时间范围 | 币种对 | 策略标签 | 盈亏方向（赢/亏/全部）
- 状态 Badge：`open` (蓝) | `closed` (绿/红)

**右栏（详情）**：选中交易的完整详情

```
┌──────────────────────────────────────────┐
│ BTC/USDT LONG — Closed +$3,960 (+12.3%) │
│ ──────────────────────────────────────── │
│ Entry:   2024-01-10 @ $64,200            │

│ Duration: 5d 2h                          │
│ Strategy: breakout-v1                    │
│ ──────────────────────────────────────── │
│ 📝 Entry Reason:                          │
│ [BTC 突破日线压力位，MACD 金叉确认...]     │
│                                           │
│ 📝 Exit Reason:                           │
│ [触达预设 TP1，保留 30% runner...]         │
│                                           │
│ 😊 Emotion / Notes:                       │
│ [执行严格按计划，没有冲动加仓...]           │
│                                           │
│ 🏷️ Tags: #breakout #btc #discipline-ok    │
│ ──────────────────────────────────────── │
│ Trade Chain:                              │

│ ──────────────────────────────────────── │
│ Notion Sync: ✅ Synced at 2024-01-15      │
└──────────────────────────────────────────┘
```

**复盘备注区**：
- 可编辑的富文本/Markdown 区域
- 入场理由、退场理由、情绪记录、错误标签
- 标签系统（可自定义标签如 `#fomo`, `#discipline-ok`, `#revenge-trade`）

**Trade Chain 可视化**：
- 时间线/步骤条样式，显示这笔交易的完整链路
- Signal → OrderIntent → Order → Fill(s) → Position Change → Close → Journal

### 4.6 Alerts（告警与通知）

**布局**：通知流 + 过滤器

**通知流**：
- 按时间倒序排列
- 每条通知包含：时间 | 类型图标 | 标题 | 描述 | 严重级别 Badge

**通知类型与图标**：
| 类型 | 图标 | 颜色 |
|------|------|------|
| TradingView 信号 | ⚡ 闪电 | 蓝色 |
| 订单成交 | ✅ 勾选 | 绿色 |
| 订单失败 | ❌ 叉号 | 红色 |
| 止损触发 | 🛑 停止 | 红色 |
| 止盈触发 | 🎯 靶心 | 绿色 |
| 风控警告 | ⚠️ 警告 | 琥珀色 |
| 系统状态 | 🔧 齿轮 | 灰色 |

**过滤器**：
- 类型多选筛选
- 时间范围
- 严重级别（Critical / Warning / Info）
- 已读/未读 切换

**空状态**：
- "No alerts. System is running smoothly."

### 4.7 Settings（系统设置）

**布局**：分组卡片式设置页

**Section 1 — General（通用）**
- 当前模式切换：`SANDBOX` ↔ `LIVE`
  - 切换到 LIVE 需要二次确认弹窗："You are switching to LIVE mode. Real money will be at risk."
- 默认账户选择
- 语言偏好（future）

**Section 2 — Exchange Account（交易所账户）**
- 已连接账户列表
- 每个账户：名称 | 交易所 | 模式 | 连接状态
- 添加新账户按钮
- 编辑/删除操作

**Section 3 — Risk Rules（风控规则）**
- 规则列表，每条规则一行：
  - 规则名称 | 当前配置值 | 开关(启用/禁用)
- 规则类型：
  - 单笔最大风险 %
  - 单笔最大仓位
  - 当日最大亏损限额
  - 必须设置止损
  - 冷静期（连续亏损后）
- 修改配置弹出编辑弹窗

**Section 4 — Notifications（通知配置）**
- Telegram 通知开关
- Telegram Bot Token / Chat ID 配置
- 通知事件选择（多选复选框）

**Section 5 — Notion Sync（Notion 同步）**
- 同步开关
- 上次同步时间
- 手动触发同步按钮
- 同步任务队列状态

## 5. 交互模式

### 5.1 确认弹窗

用于高危操作（切换 LIVE 模式、Force Override 风控、一键平仓）：

```
┌──────────────────────────────────────┐
│  ⚠️ Confirm Action                   │
│                                      │
│  You are about to close all          │
│  BTC/USDT positions (0.5 BTC).       │
│                                      │
│  Current P&L: +$3,960.00            │
│                                      │
│  Type "CONFIRM" to proceed:          │
│  [                    ]              │
│                                      │
│        [Cancel]    [Confirm]         │
└──────────────────────────────────────┘
```

- 高危操作需要输入 "CONFIRM" 文字确认
- 中危操作使用简单二次确认按钮

### 5.2 Toast 通知

- 右上角浮层
- 成功 = 绿色左边框
- 错误 = 红色左边框
- 警告 = 琥珀色左边框
- 3-5 秒自动消失，可手动关闭

### 5.3 空状态

所有列表/表格都需要空状态设计：
- 简洁的图标 + 一行文字说明
- 可选的引导操作按钮
- 不要使用大面积插画（保持专业感）

### 5.4 加载状态

- 表格/列表：Skeleton 骨架屏
- 操作按钮：Spinner + 禁用状态
- 全局数据刷新：TopBar 中显示微妙的加载指示器

## 6. 关键设计原则总结

1. **信息密度 > 美观性**：这是工作台，不是营销页面。数据要密，留白要克制。
2. **数字必须整齐**：所有金融数字使用等宽字体，小数点对齐，红绿色明确。
3. **模式感知**：SANDBOX/LIVE 模式必须始终可见，LIVE 模式要有"真钱在线"的紧张感。
4. **风控视觉化**：风控检查结果是产品灵魂，必须清晰展示每条规则的通过/拒绝/警告。
5. **审计链路**：交易链路（Signal → Intent → Order → Fill → Journal）要可视化可追溯。
6. **深色优先**：默认深色主题，降低长时间使用的视觉疲劳。
7. **最小交互层级**：Dashboard 上的关键信息不要需要展开/折叠，直接摆出来。
8. **操作确认**：任何涉及真金白银的操作（下单、平仓、切换 LIVE 模式）必须有确认步骤。

---

## 7. 技术约束

- **前端技术栈**：Next.js + TypeScript + Tailwind CSS + shadcn/ui
- **图标库**：Lucide Icons
- **图表库**：轻量级图表（如 recharts 或 lightweight-charts）
- **响应式**：桌面优先设计（最小宽度 1280px），不需要移动端适配
- **无障碍**：基本键盘导航支持，但不作为 V1 优先级

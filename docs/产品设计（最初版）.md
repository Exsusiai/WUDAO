# 悟道 - 个人交易终端软件

##  产品概述
“悟道”是一个面向个人投资者（即我个人）的交易终端软件。该软件同时提供以下功能：
1. 面向我个人日常的交易使用
2. 提供一系列接口，用于对接我的私人交易 AI Agent助理使用
### 为什么要有这个软件？
既然市面上已经有了这么多交易软件，而且很多券商自己也提供丰富的功能，这个软件的核心需求主要有两个：
1.  **作为个人交易者，我需要一个终端面板去监督自己的纪律，严格执行交易**。
    设计一些比券商软件默认功能更复杂的仓位管理和止盈止损功能。
2.  **我希望它能与我的交易 AI Agent 深度耦合**。
    这样，我和我的 AI Agent 就能有一个共同管理和测试交易的面板。我能更直观地了解我的交易，以及我的 AI Agent 所进行的交易和它所监视的策略。

## 功能设计
产品主要包括以下功能
### 交易功能
- 连接 TradingView：我在 TradingView 上通过手动操作进行的交易，可以将其交易信号实时回传给本终端软件，并由本终端软件发送指令到实际的交易软件或券商
- 连接券商：本终端软件可以直接连接到我使用的券商平台，进行交易指令的发送和订单管理
- 自动止盈止损：
根据我的交易计划设置自动的止盈止损点，然后通过从券商那边回传获得的交易信号，去实现一些比券商自带功能更高级的止盈止损机制。

### 交易记录
本终端软件会记录所有的交易活动，包括交易信号、订单状态、成交记录等，方便我进行后续的分析和总结（交易记录可能储存在数据库，或者像 Notion 这类文本笔记软件中）

### 纪律执行
- 交易提示面板：交易者可以写一些督促自己交易的自定义内容，并放在面板中合适的位置。
- 自定义交易纪律，并将其量化为可通过软件监控的方式，从而由软件严格执行。
- 每一笔交易在正式向券商软件下单之前，都会通过纪律执行部分判断其是否符合交易纪律；如果不符合，将被直接拦截。

### 策略管理，测试和回测
- 策略的设计：
由我和 AI Agent 共同提交，涵盖以下多个方向：

1. 宏观与新闻策略
   (a) 基于量化指标的策略
   (b) 根据新闻、板块消息或宏观信号制定的策略
2. 链上追踪策略
   (a) 监控链上“巨鲸”动态
   (b) 关注特定账号并跟踪其动向进行交易
3. 数据驱动与套利策略
   (a) 从已有数据平台获取的策略
   (b) 通过价差、平台差异、汇率等进行的套利策略
   (c) 网格交易策略
4. 特定平台策略
   针对 Polymarket 等平台的专项策略

- 策略管理
我希望建立一个策略管理器，对上述这些涵盖范围极广的策略进行统一管理和测试。这个管理器应该能够支持策略的版本控制、参数调整、性能评估等功能。
并且由于不同策略可能面向不同标的，可以考虑针对这些策略，设计它们对应的标的，从而让策略在特定的标的上自动运行。

- 回测
这部分功能可以考虑直接连接 TradingView 或其他外部工具来实现，尽量不要自己从零造轮子。

- 交叉验证
对于一些策略，尤其是基于数据驱动的策略，我希望能够进行交叉验证，以评估其在不同市场环境下的表现。这可以帮助我更好地理解策略的稳健性和适用范围。

## 面板/UI设计
面板分为几大块：

1. 提示区域
   有一个地方用于显示我自己写给自己的话，用来在做交易时提醒自己。

2. 持仓显示
   显示我的持仓情况，并且针对这些持仓，要有一套非常高级的止盈止损策略和仓位管理策略来进行管理。

3. 现有交易策略管理
   我希望在交易策略这一块，能有一个多维度的筛选器和管理工具，以便实现一些比较复杂的策略功能。

## 功能补充说明
TradingView 内部自带回测和各种策略管理功能，因此在我们的交易软件内部，就不需要重点去实现这些功能，而是尽可能去使用别人造好的轮子。

我们的交易软件重点应该在于服务我们自己的一些需求，比如交易软件没有实现的高级止盈止损，或者是自动计算仓位的功能。

核心功能需求如下：
1. 自动计算功能
   (a) 基于单笔交易最大可承受损失、最大杠杆倍数，以及止盈止损点位，自动计算开仓金额等。
2. 自动化利润管理。可以手动配置一部分，剩下的自动计算
   (a) 获得利润后自动保证利润。
   (b) 自动移动止损线。

这些是我们交易系统面向人类用户的一些核心功能。

3. 自动化交易记录
   还有就是每次交易完，系统会自动向数据库中写入我们的交易记录。这非常便于我们记录自己的交易情况，并进行后续复盘。

4. 交易纪律框架

我们可以考虑把每一条交易纪律都通过程序化的方式实现。在真正交易的时候，可以通过激活或者禁止某些交易纪律的方式，强制程序在每次交易前都进行一次检查。如果检查不通过，就不能开单。

## 技术框架
- 前端：React + TypeScript
- 后端：考虑延迟较低、性能较高的框架，以保证我们进行程序处理的时候，不会影响我们的交易速度

## 产品特点
这个产品除了服务于我以外 这个产品本身如果要对外提供的话 它的特点或者说卖点是什么？
就如他的名字 “悟道”一样 这个产品的核心卖点在于它能够帮助用户更好地理解和执行交易纪律
市场上已经有很多高级且专业的交易软件 没有必要再去竞争专业性
我们产品的用户可以专注于个人交易者 这个细分市场 提供一些更适合个人交易者的功能和体验
可以考虑：那些成功的交易者他们的交易纪律是什么？他们是如何执行这些纪律的？我们可以把这些成功交易者的经验总结出来，并通过我们的软件来帮助用户更好地理解和执行这些纪律。
从交易中我们又能学到哪些人生的道理？
